ssh based file system SSHFS

Mount a remote server directory through the web

Using an SSH, you can establish a connection to a directory on a remote host. The directory can be used to make backups to it and use it as a safe storage.

When combining it with disk encryption on both servers, all data can be kept safe and secure.

Disadvantages may be low(er) speed and the need for this file system to be reliably connected to the target.

Make sure key permissions are set right

sudo chown ss:ss /home/ss/.ssh/id_ecdsa
sudo chmod 400 /home/ss/.ssh/id_ecdsa

Add an SSH identity file to the systemwide SSH config so it can be used. Note sshfs is run as root!

sudo vi /etc/ssh/ssh_config

    Host node2.wurstbot.com
            IdentityFile /home/ss/.ssh/id_ecdsa

Make and mount mountpoint

.. Using compression ( -C ) and port 3200 ( -p )

cd
mkdir mountpount

sudo sshfs -o allow_other -p 3200 admin@node2.wurstbot.com:/home/admin /home/ss/mountpoint -C

Umount it

sudo umount mountpoint
