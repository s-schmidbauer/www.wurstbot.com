Now using HTTPS only

From today on, wurstbot supports secure and encrypted connections only. All HTTP connections will be redirected to HTTPS. This will ensure secure reading and writing.

As you can see, we're using very high security standards with TLS 1.2 and ECDSA SSL key exchange. I've also reworked the SSL configuration and header config.

You can read more about in the blog under "Apache config hardening"

Also, secure uploading in the near future ;-) Looking forward hearing from you.
